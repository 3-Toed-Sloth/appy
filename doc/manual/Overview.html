1 Overview

This section provides an abstract overview of the library. A complete list of all classes and methods can be found in section Reference.

Appy is an object-oriented framework whose main classes are application% and gui-application% for building command line and GUI applications respectively.

1.1 Main Features

Creating and packaging a self-contained end-user application in Racket requires a lot of boring detail that varies only slighly between applications. Appy helps you with the following tasks:

  • Internationalization: The lang module exports functions for associating English format strings with symbolic keys using the defloc macro and a function tr that takes a key and optional format paramaters and returns the localized string according to the current language setting set by load-language. The default language is English. Templates can be created for other languages at compile-time and translations may be added at runtime and compile-time.

  • Error Handling: The err module exports loggers and convenience functions for specifying localized error messages. While writing a module, you register numeric keys and corresponding translation keys for the error messages using register-errors and use one of the error functions with the numeric key to raise the error. It is your job to ensure that each module has its own range of numeric error keys. While this system might appear cumbersome at first, it has the advantage that all error messages are localizable and therefore end-user readable by default and that numeric error codes can be used for further reference in a manual.

  • Preference Handling: The prefs module exports functions for associating symbolic keys with preferences and saving and loading them from the place where they usually reside according to operating system conventions. Once the preference system has been initialized and default preferences have been defined with defpref, serializable Racket values can be stored and retrieved with the pref function. Preferences are stored in an Sqlite database that is handled automatically behind the scenes by Appy.

  • Commands & Undo Management: The commands module handles a database-driven undo manager with disk-based, unlimited undo and redo functionality and provides functions and classes for defining commands that perform some function and are automatically managed by the Undo Manager. The disk-based method is slow and not suitable for every type of application but if you need the functionality it’s there.

  • An Easy-to-use Database: The storage module provides a storage% class that wraps an Sqlite database into an easy-to-use object-oriented API. If you need a simple key-value store or some way to store a few tables with columns without complicated queries, this class might come in handy. For more complex data-driven models it might be more advisable to use Racket’s full database capabilities and SQL.

  • Event-Subscriber System: The notify module provides the means to implement the notify-subscriber pattern for easy communication between distant system components. Depending on how you use it, this may be a curse or blessing, but it greatly simplifies implementing the model-view-controller GUI pattern. Check out the Notes example for a use of this module.

  • Platform-specific Abstractions: The platform module contains some platform-specific abstractions that are used by other modules. Generally speaking, Appy aims at providing platform-compliant defaults for file paths, icons, and other differences between platforms. Supported platforms are macOS, Windows, and Linux.

  • Easy Deployment: The deploy module allows you to easily deploy an application once it has been set up. It expects a method run to start the application and the parameter the-application to be set to an instance of application%, from which (deploy) pulls all the necessary meta-information to compile the application and create a packed distribution file. Pre and post compilation scripts can be provided to e.g. upload the distribution to a server.

1.2 Importing the Library

 (require appy) package: appy
This imports the bindings for applications, storage, preferences, custom error handling, and internationalization but not any GUI-related functionality.

 (require appy/gui) package: appy
This imports all the bindings from appy and in addition the support for building GUI desktop applications using Racket’s GUI framework. For cross-platform graphical end-consumer applications, this is the right choice.
